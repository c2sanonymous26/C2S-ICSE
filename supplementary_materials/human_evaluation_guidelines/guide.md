# Task Description

This evaluation requires you to review constraints generated by two anonymized automated methods. 

The goal is to assess the quality of each constraint. For each constraint, please inspect its provided description and implementation, then assign one of three levels, Level A, Level B, or Level C, based on the following two questions:

- **Q1. Is the constraint interpretable, and does it express a substantive environmental condition?**
- **Q2. Is the expressed condition physically plausible in the current scenario?**

Please score each constraint by assigning a level according to the following criteria:

- **Level A**: The constraint is interpretable, expresses a substantive environmental condition, and the condition is physically plausible in the current scenario.
- **Level B**: The constraint largely satisfies the two questions above, but has minor issues or some uncertainty.
- **Level C**: The constraint fails either question: it is not interpretable, does not express a substantive environmental condition, or expresses a condition that is not physically plausible in the current scenario.

Each constraint is provided with two files:

**`constraint.json`**: contains the natural-language semantics, logical structure, and predicate descriptions of the constraint.

**`bfuncs_*.py`**: contains the executable Python implementation of the predicates.

Some numeric thresholds in `bfuncs_*.py` may appear as fractions. Please interpret them by their numeric value rather than by their textual form.

# Evaluation Procedure

Please complete the evaluation using the following steps.

## Step 1: Read the background materials

Before evaluating the constraints, please read the following two background files in the `background/` directory:

- **`scenario.md`**: the statement for a taxi-management scenario.
- **`fields.md`**: the field description. It lists the meaning, type, and unit of each context field.

## Step 2: Read the examples

Before scoring the constraints, please read **`example.md`**. It provides example constraints for Level A, Level B, and Level C, together with the scoring reasons. These examples are provided for reference.

## Step 3: Evaluate each constraint

For each constraint, read the files in the following order:

1. First read the `semantics` field in `constraint.json` to inspect the natural-language description of the constraint.
2. Then inspect the `structure` field and the `bfuncs` entries to review the logical structure and the concrete predicate implementations. You may also refer to `bfuncs_*.py` for the executable code.
3. Based on the above information, judge whether the constraint answers Q1 and Q2. Then fill in the score and reason in the questionnaire.

Please write a short reason in the comment column. For constraints scored as B, please clearly state the concern.
